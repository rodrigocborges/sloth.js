Move slow. Build fast. 🦥

Sloth.js is a lightweight, declarative frontend toolkit designed for developers who want to write less boilerplate and focus on logic. It bundles DOM manipulation, reactive state, form handling, routing, and HTTP requests into a single, zero-dependency module.

🔌 Zero Config

Drop it in. No build step required. Works natively with ES modules.

⚡ Declarative

Bind data, trigger actions, and handle forms with simple HTML attributes.

📦 All-in-One

Fetch wrapper, DOM utils, router, validator, and reactivity in ~6KB.

Installation

Include Sloth.js directly in your HTML. It exports a default singleton instance.

<script type="module">
  import app from './sloth.js';
  // app is now ready to use
</script>

DOM Utilities Chainable

Lightweight jQuery-like helpers built on native DOM APIs.

app.el(selector) & app.els(selector)

Select single or multiple elements and chain methods.

app.el('#title').text('Updated').addClass('active');
app.els('.card').on('click', (e) => console.log(e.target));
MethodDescription
.html(val)Get/set innerHTML
.text(val)Get/set textContent
.addClass()Add CSS classes
.removeClass()Remove CSS classes
.toggleClass()Toggle CSS classes
.on(event, handler)Attach event listeners

HTTP Client Fetch Wrapper

Simplified async requests with automatic JSON parsing and auth headers.

app.setBaseURL('https://api.example.com');
app.setToken('your_jwt_token');

const users = await app.get('/users');
const newUser = await app.post('/users', { name: 'Alex' });

Available methods: get, post, put, delete. All return Promises. Errors are logged automatically.

Declarative Actions

Bind JavaScript functions directly to HTML elements using data-click and data-submit.

<!-- HTML -->
<button data-click="sayHi">Greet</button>
<form data-submit="saveUser">
  <input name="email" type="email" />
  <button type="submit">Save</button>
</form>
// JS
app.actions({
  sayHi: ({ el, params }) => {
    alert(`Hello! Params: ${params.join(', ')}`);
  },
  saveUser: ({ data }) => {
    console.log('Form payload:', data);
  }
});
Note: Pass parameters like data-click="sayHi('John', 25)". Form submissions automatically include data as a JSON object.

Auto-Bind State Reactive

Create a reactive proxy that syncs with the DOM via data-bind attributes.

const state = app.bind({ user: { name: '', age: 20 } });

// Update DOM automatically
document.getElementById('nameInput').addEventListener('input', e => {
  state.user.name = e.target.value; // Triggers re-render on [data-bind="user.name"]
});

Any element with data-bind="path" will update its value (inputs) or textContent (divs/spans) when the state changes.

Template Engine

Simple string interpolation with nested property support.

const html = app.render(
  '<h2>{user.name} ({user.age})</h2><p>{status}</p>',
  { user: { name: 'Rodrigo', age: 30 }, status: 'Active' }
);
// → <h2>Rodrigo (30)</h2><p>Active</p>

SPA Hash Routing

Lightweight hash-based router for single-page applications.

app.route('/', () => app.el('#app').html('<h1>Home</h1>'));
app.route('/user/:id', ({ id }) => app.el('#app').html(`User ${id}`));
app.route('/about', () => app.el('#app').html('<h1>About</h1>'));

app.startRouter();

Navigate with #/, #/user/42, etc. Parameters are parsed and passed to the handler.

Forms & Validation

app.formToJSON(form)

Converts form inputs to a clean JSON object. Handles checkboxes, radios, and multi-selects automatically.

app.validate(form)

Validates inputs using HTML data-* attributes. Returns { valid: boolean, errors: Object }.

<input name="email" data-required data-email />
<input name="age" data-required data-min="18" data-max="99" />
<input name="code" data-pattern="^[A-Z]{3}$" />
const { valid, errors } = app.validate('#myForm');
if (!valid) console.error('Validation failed:', errors);

Global Loader

Show/hide a fullscreen loading overlay. Customizable via HTML.

app.setLoader('<div class="loader">🦥 Loading...</div>');
app.showLoader();
// ... async operation ...
app.hideLoader();

Event Delegation

Attach listeners to parent containers. Automatically handles dynamically added elements.

app.on('.dynamic-btn', 'click', (e, target) => {
  console.log('Clicked:', target.id);
});
app.off('.dynamic-btn', 'click'); // Remove specific or all
Why delegation? One listener handles all matching elements, reducing memory overhead and simplifying dynamic DOM management.
